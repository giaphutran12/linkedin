managing 5 projects without burning out is less about discipline
and more about protecting your context

that's the biggest thing i'm learning right now

when i get fried
it's usually not because i worked too hard

it's because i let too many threads mix together in my head

so this is what helps me

1. keep prompts SHORTER than you want

i use speech-to-text a lot
which means i can ramble fast

that's dangerous

if the prompt gets too long
the agent starts solving the wrong thing

even good models drift when your instructions drift

2. move SEQUENTIALLY

if i'm answering questions on one task
i do NOT switch to check another finished thread halfway through

that sounds small
but context switching kills flow fast

you feel productive
while getting dumber every 5 minutes

3. treat the plan like SOURCE CODE

before implementation starts
i skim the plan like i'm reviewing a PR

if i don't understand a word
i ask again

sometimes literally
explain it to me like i'm 5

if i don't catch the wrong assumption there
the build goes in the wrong direction
and that's on me

AI coding gets way less stressful
once you stop treating prompts like magic

it's just another system

bad input
bad output
