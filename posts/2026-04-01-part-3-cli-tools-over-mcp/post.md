until they eat your context window alive

i used to install a ton of them

looked powerful
felt sophisticated

but before the first real prompt
i had already burned a stupid amount of tokens on setup and tool context

now i default to CLI tools first

and only use MCP when i actually NEED it

that one change saved me something like 10k to 15k tokens
before the work even started

which matters a lot more when you're juggling multiple agents
multiple projects
and long planning threads

the bigger lesson is this

agent workflows don't just break because the model is weak

they break because your setup is bloated

too many tools
too much context
too much noise before the task even begins

more tooling does NOT automatically mean more leverage

sometimes it just means more drag

CLI first
MCP when necessary
